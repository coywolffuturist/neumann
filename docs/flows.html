Python router pipeline — THE top-of-funnel for all prompts. Owns interview + plan + decompose + route as one engine. Lucid shells out to `python -m neumann.router.cli` for mission routing. PR #48 (2026-05-03) shipped the Decomposer-before-Planner ordering.

Generated 2026-05-13 · This file is read-only documentation; flows.json is the canonical machine-readable form.

Pipeline Ordering

Prompt → ShapeClassifier (advisory) → Interviewer (opt-in) → ConfirmedIntent → Decomposer (intent → N sub-intents) → Planner (1+ tasks per sub-intent) → TaskDecomposer (task-level split for oversized) → per-task routing (TaskClassifier → ContextResolver → PersonaSelector → fallback → validate) → RoutingTrace[] → Fusion Todo column

Flows

  • Full mission pipeline
    triggerfnr mission "<prompt>" --json
    steps
    [
  "ShapeClassifier \u2014 advisory: single-task vs mission via regex rules",
  "Interviewer (opt-in) \u2014 LLM clarification loop \u2192 ConfirmedIntent; validate_intent() deterministic gate (target_repo regex, success_criteria non-empty, human_approved=True)",
  "Decomposer \u2014 if intent exceeds thresholds (max_lines_estimate=500, max_files_changed=3, max_distinct_outputs=2), split into N sub-intents + integration intent",
  "Planner \u2014 runs ONCE PER SUB-INTENT; emits 1+ PlannedTasks each (LLM-backed or MockPlanner)",
  "TaskDecomposer \u2014 post-Planner: split oversized PlannedTasks into N child tasks + integration task",
  "Per-task routing: TaskClassifier \u2192 ContextResolver \u2192 PersonaSelector \u2192 fallback \u2192 validator",
  "Output: PipelineResult with shape, confirmed_intent, plan, routes[]"
]
    outcomeRoutingTrace[] ready to emit into Fusion Todo column
  • Fast path (single-task, no interviewer)
    triggerfnr task "<prompt>" --json
    steps
    [
  "ShapeClassifier \u2192 SINGLE_TASK",
  "No interviewer wired \u2192 synthesize ConfirmedIntent from prompt",
  "Skip decompose + plan; route raw prompt directly as PlannedTask"
]
    outcomeSingle PersonaDecision; preserved for tests and trivial prompts
  • Phase 2 QA Watcher (FusionWatcher daemon)
    triggerExternal cron polls Fusion's In Review column every 30s
    steps
    [
  "GET /api/tasks (filter column==in-review)",
  "For each task: GET /api/tasks/{id} \u2192 PROMPT.md body",
  "QAExecutor.execute(QATask) \u2192 QAResult (verdict: PASS|FAIL|SKIP|PLANNER_BUG)",
  "RetryPolicy.decide(verdict, attempt) \u2192 RetryAction (DONE|RETRY|PAUSE_ESCALATE)",
  "Dispatch: DONE+PASS \u2192 move to done silently. DONE+SKIP \u2192 done + breadcrumb. RETRY \u2192 comment + move back to in-progress. PAUSE_ESCALATE \u2192 comment + /pause + WhatsApp notify via clawdbot"
]
    outcomeQA gate is independent of Coder; idempotent via WatcherState

Invariants

  • Interviewer is opt-in (CLI/Slack/Lucid/web wire one; offline/tests leave None → skip interview)
  • Fast path preserved: SINGLE_TASK + no interviewer → bypass decompose/plan
  • Decompose BEFORE Planner (keeps Planner input small; Brendan's design)
  • Task-decompose AFTER Planner (catches oversized tasks Planner emits; FN-002)
  • Validation gate deterministic (no LLM; schema rules; max 8 rounds)
  • ConfirmedIntent approval explicit (human_approved=True before Planner runs)
  • QA watcher idempotent (WatcherState outside Fusion; survives restarts)
  • Personas extensible (registry loads presets + custom JSON at runtime)
  • FALLBACK_SENTINEL is a dispatch signal; fallback.py resolves at runtime
